• null 代表 沒有值存在 (the absence of a value)
  • typeof null === "object" // => true
  • null 自成一型別，而他自己是這個型別的唯一成員，他可被用來代表數字 字串或物件中的 沒有值 (no value)
  • JavaScript 還有一個代表 值不存在 的值：undefined
  • undefined 代表更深刻的不存 (absence)
  • undefined case:
    • 尚未初始化的變數
    • query 不存在的的物件特性或陣列
    • 沒有回傳值的函式也會回傳 undefined
    • 沒有給 argument 的函式 parameter 值
  • typeof undefined === "undefined" // => true
  • null == undefined // => true
  • null === undefined // => false
  • undefined 視為代表系統層級的 非預期的 類似錯誤的 '沒有值'
  • null 代表程式層級的 正常的 預期中的 '沒有值'
  • 如果要 assign 給變數或特性 property 貨船值給函式， null 幾乎總是正確的選擇
